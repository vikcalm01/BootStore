# BootStore — система учёта обуви

Веб-приложение на Flask для учебного учёта товаров и заказов с разграничением прав по ролям.

## Что умеет система
- Авторизация по ролям: `guest`, `manager`, `admin`.
- Просмотр каталога товаров для всех ролей.
- Поиск/фильтрация/сортировка товаров для менеджера и администратора.
- CRUD товаров для администратора.
- Просмотр заказов для менеджера и администратора.
- CRUD заказов для администратора.

---

## Быстрый старт

```bash
python3 -m pip install -r requirements.txt
python3 scripts/init_db.py
python3 src/webapp.py
```

Открыть: `http://127.0.0.1:5000/login`

## Инструкция пользователя

### 1) Вход в систему
- Перейдите на страницу `/login`.
- Можно войти как зарегистрированный пользователь (email + пароль) или как гость.

### 2) Работа с товарами (`/products`)
- **Гость**: только просмотр каталога.
- **Менеджер**: просмотр + поиск + фильтр по поставщику + сортировка по остатку.
- **Администратор**: всё как у менеджера + добавление/редактирование/удаление товаров.

### 3) Работа с заказами (`/orders`)
- **Менеджер**: просмотр заказов.
- **Администратор**: просмотр + добавление/редактирование/удаление заказов.

### 4) Выход
- Кнопка **«Выход»** в верхнем меню очищает сессию и возвращает на страницу логина.

## Что делает каждый компонент

### Backend
- `src/webapp.py` — основной Flask-приложение: роуты, авторизация, проверки ролей, операции с БД.

### Шаблоны
- `templates/base.html` — общий каркас страниц (header, меню, flash-сообщения).
- `templates/login.html` — страница входа.
- `templates/products.html` — таблица товаров и панель фильтров.
- `templates/product_form.html` — форма создания/редактирования товара.
- `templates/orders.html` — таблица заказов.
- `templates/order_form.html` — форма создания/редактирования заказа.

### Стили и медиа
- `static/styles.css` — единый стиль интерфейса.
- `assets/` — изображения товаров и логотипы.

### Данные и служебные скрипты
- `app.db` — SQLite база данных.
- `scripts/init_db.py` — создание схемы и начальных данных.
- `scripts/show_users.py` — просмотр пользователей из БД.
- `scripts/generate_algorithm_pdf.py` — генерация PDF из раздела docs.
- `docs/database_schema.sql` — SQL-схема базы.

## План презентации проекта

### Слайд 1 — Проблема и цель
- Что автоматизируем: учёт обуви и заказов.
- Почему это важно: скорость, меньше ошибок, разграничение доступа.

### Слайд 2 — Архитектура
- Flask + Jinja2 + SQLite.
- Поток данных: UI → маршруты → SQL → UI.

### Слайд 3 — Роли и сценарии
- Guest / Manager / Admin.
- Кратко показать, какие действия доступны каждой роли.

### Слайд 4 — Демонстрация UI
- Логин.
- Каталог товаров (поиск, фильтр, сортировка).
- Таблица заказов.

### Слайд 5 — Админ-функции
- Добавление/редактирование/удаление товара.
- Добавление/редактирование/удаление заказа.

### Слайд 6 — Надёжность данных
- Ограничения БД, внешние ключи.
- Проверка валидности на сервере.

### Слайд 7 — Итоги и развитие
- Что уже готово.
- Что можно улучшить: хеширование паролей, пагинация, API, тесты.
